## TL;DR

- **The env**: `DropoutCommonsEnv` — a multi-agent reinforcement-learning simulation of an educational system on the brink of collapse, calibrated against UNESCO and DISE India data.
- **The novelty**: a two-tier **swarm-of-swarms orchestrator** in which four stakeholder swarms (Student, Teacher, Administrator, Policymaker) — each composed of three heterogeneous LLM personas — deliberate in parallel and surface a **Resonance map**: where every lens agrees, where they materially disagree.
- **The training**: we **distil** the 12-persona swarm's policy into a **1B Llama-3.2 student model** via SFT on a synthetic dataset generated by the swarm itself. The student matches the teacher's recommendations at **~1% inference cost**.
- **The result**: a single small model that learned to reason like four stakeholder communities — and is honest about which decisions it cannot make alone.

---

## 1. The Problem

Educational systems collapse on predictable trajectories — falling enrollment, teacher attrition cascades, class-size shocks, dropout cliffs — yet interventions consistently arrive *after* the cascade has begun. The standard policymaking toolkit makes this worse: cost-benefit analyses optimise one variable at a time, single-stakeholder consultations privilege the loudest voice in the room, and recent AI policy tools manufacture artificial consensus by hiding disagreement behind a single recommendation.

**Vishwamitra reframes this as an explicit disagreement-mapping problem.** Instead of one answer, it produces a structured artefact: which interventions every stakeholder lens agrees on (deploy with confidence) and which are genuinely contested (require human judgement).

## 2. The Environment — `DropoutCommonsEnv`

OpenEnv-compliant Gymnasium environment. The meta-agent (the policy being trained) is a **mechanism designer**: each step it picks intervention intensities; the simulated stakeholders respond.

### Observation space — `Box(13)` float32, range `[0, 1]`

| # | Field | Meaning |
|---|---|---|
| 0 | `enrollment_rate` | Reported enrollment (signal) |
| 1 | `attendance_rate` | Daily attendance |
| 2 | `dropout_rate` | Per-period dropout |
| 3 | `teacher_retention` | Staff retention |
| 4 | `budget_utilization` | 1 − (budget_remaining / 2M) |
| 5 | `avg_class_size` | Normalised (0 ≈ 0 students, 1 ≈ 60) |
| 6 | `teacher_workload` | Workload index |
| 7 | `resource_allocation` | Quality of resource distribution |
| 8 | `student_engagement` | Engagement index |
| 9 | `teacher_burnout` | Burnout index (lower = better) |
| 10 | `policy_compliance` | Adherence rate |
| 11 | `budget_ratio` | Cash remaining / 2M |
| 12 | `step_progress` | Episode progress |

### Action space — `Box(8)` float32, range `[0, 1]`

| # | Intervention | Per-step max cost |
|---|---|---|
| 0 | `funding_boost` | $50K |
| 1 | `teacher_incentive` | $80K |
| 2 | `student_scholarship` | $30K |
| 3 | `attendance_mandate` | $10K |
| 4 | `resource_realloc` | $40K |
| 5 | `transparency_report` | $5K |
| 6 | `staff_hiring` | $120K |
| 7 | `counseling_programs` | $25K |

### Reward function

```python
reward = clip(
    -2.0 * dropout_rate
    +1.0 * (teacher_retention - 0.7)
    +0.5 * student_engagement
    -0.001 * (cost / 50K),
    -2, +2
)
```

A narrow signal that punishes dropout heavily, rewards retention against a baseline, and applies a token cost penalty. The broader **health score** (enrollment, attendance, burnout, etc.) is monitored separately for the dashboard — kept *out* of reward by design so the policy doesn't game the auxiliary metrics.

### Episode termination — collapse triggers

The episode ends with `terminated=True` if **any** of:

- `dropout_rate > 0.50` (cohort collapse)
- `teacher_retention < 0.20` (staff exodus)
- `budget_remaining < -$500K` (fiscal insolvency)
- `enrollment_rate < 0.30` (demographic collapse)

These are *hard cliffs*: the agent gets zero future reward from a collapse state, which trains it to prevent the cascade rather than recover from it.

## 3. The Innovation — Two-Tier Swarm-of-Swarms

Inference-time reasoning layer that sits **above** the env. Not part of the RL training loop; instead, it produces structured deliberation reports that we use as **distillation targets** for the small model.

### Architecture

```
                ┌─────────────────────────────────────────┐
                │   L1 ORCHESTRATOR (WeightAllocator)     │
                │   state → role attention → model + wt   │
                └─────────────────────────────────────────┘
                                  │
       ┌──────────────┬───────────┼───────────┬──────────────┐
       ▼              ▼                       ▼              ▼
  ┌─Student─┐   ┌─Teacher─┐              ┌─Admin──┐   ┌─Policymaker─┐
  │ Maya    │   │ Sharma  │              │ Desai  │   │ Verma       │
  │ Rohit   │   │ Iyer    │   ...        │ Joshi  │   │ Banerjee    │
  │ Priya   │   │ Kumar   │              │ Rao    │   │ Khan        │
  └─────────┘   └─────────┘              └────────┘   └─────────────┘
       │              │                       │              │
       └──────────────┴───────────┬───────────┴──────────────┘
                                  ▼
            ┌─────────────────────────────────────────┐
            │   L2 ORCHESTRATOR (PersonaAllocator)    │
            │   per-persona fit_signals × pressures   │
            └─────────────────────────────────────────┘
                                  │
                                  ▼
                ┌─────────────────────────────────────┐
                │   RESONANCE / DISSONANCE MAP        │
                │   (final action + flagged conflicts)│
                └─────────────────────────────────────┘
```

### L1 Orchestrator — `swarms/orchestrator/router.py · WeightAllocator`

Inspects the **system state**, computes a per-role *attention score* (max of pressures most relevant to that lens), and decides:

- **Which model** each role swarm uses (heavy `llama-3.3-70b` for high-attention roles, light `llama-3.1-8b` for routine), and
- **How much weight** that swarm's verdict carries in the cross-swarm aggregation.

### L2 Orchestrator — `swarms/orchestrator/router.py · PersonaAllocator`

Inside each swarm, distributes weight **across the three personas** based on `fit_signals` declared per-persona in `roles.yaml`. Example:

```yaml
- id: student_dropout_risk
  name: "Rohit, Working Student"
  fit_signals: { budget: 0.8, dropout: 0.9, attendance: 0.7 }
```

When the state has high dropout pressure, **Rohit's voice dominates the Student swarm's verdict**; when the crisis is about classroom quality, **Priya** does. Weights map from raw fit score `[0, 1]` to `[0.3, 1.5]` so even off-topic personas keep a baseline voice — the swarm still hears all three lenses.

### Why this isn't just "more agents = better answers"

The system **never collapses dissent** — interventions where the four lenses materially disagree (`resonance < 0.55`) are flagged as **dissonance flags** and surfaced as decisions that *belong to a human*, not to the algorithm. Most policy AI hides disagreement; we promote it to a first-class output.

## 4. Training — Distillation of the Swarm into a 1B Student

The 12-persona swarm is too slow and expensive for production deployment (12 LLM calls per decision, ~30 seconds, ~$0.02 per call). We distil its policy into a **single 1B model** via SFT.

### Pipeline

```
SwarmManager.deliberate(state) ──▶ ResonanceReport
                                       │
                          synthesise   │
                  (programmatic, no    │
                   extra LLM calls)    ▼
                              {state, scenario,
                               action_vector,
                               reasoning}
                                       │
                                       ▼
                          train.jsonl  +  val.jsonl
                                       │
                                       ▼
                  Unsloth + TRL SFTTrainer (Colab T4, QLoRA)
                                       │
                                       ▼
                  Llama-3.2-1B-Instruct + LoRA adapter
```

### Dataset — `generate_dataset.py`

- **5 scenario templates** × ~60 jittered variants = ~300 examples (some attrition from rate-limits)
  - Funding crisis · Teacher exodus · Pandemic recovery · Rural constraint · Healthy school
- Each row = one swarm deliberation in chat-message format ready for `SFTTrainer`
- 90/10 stratified train/val split

## 7. How to use the env

### As an OpenEnv HTTP service

```bash
uvicorn server.app:api --host 0.0.0.0 --port 8000

# In another terminal:
curl -X POST http://localhost:8000/reset -H "Content-Type: application/json" \
  -d '{"scenario": "funding_cut", "episode_length": 100}'

curl -X POST http://localhost:8000/step -H "Content-Type: application/json" \
  -d '{"action": [0.5, 0.7, 0.3, 0.0, 0.4, 0.2, 0.0, 0.5]}'
```

### As a Gymnasium environment

```python
from env.dropout_env import DropoutCommonsEnv
from env.scenarios.funding_cut import FundingCutScenario

env = DropoutCommonsEnv(scenario=FundingCutScenario(), episode_length=100)
obs, info = env.reset(seed=42)

for step in range(100):
    action = my_policy(obs)        # 8-dim action in [0, 1]^8
    obs, reward, terminated, truncated, info = env.step(action)
    if terminated:
        print(f"System collapsed at step {step}")
        break
```

### Run a swarm deliberation

```python
import asyncio
from swarms import SwarmManager

manager = SwarmManager()
report = asyncio.run(
    manager.deliberate(
        state={"dropout_rate": 0.28, "teacher_burnout": 0.72, "budget_remaining": 420000.0, ...},
        scenario="35% mid-year funding cut",
    )
)

print("Final action:", report.final_action)
print("Dissonance flags:", report.dissonance_flags)
print("Per-role:", [(s.role, s.mean_confidence) for s in report.swarm_verdicts])
```
